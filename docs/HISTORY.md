# 📜 AFP // Project Wiki & History

This document chronicles the evolution of the **AI Forge Protocol (AFP)** and the **TestForge** standard.

## 🚀 The Journey

### Phase 1: The Core Protocol (May 2026)
- **Initiation:** Defined the basic `TestForgeAdapter` contract.
- **Harness Scaffolding:** Developed the initial Python execution engine (`harness.py`).
- **Standardized Metrics:** Integrated basic semantic entropy and consistency scoring.

### Phase 2: Cognitive Integration (Early June 2026)
- **Council of Seven Wiring:** Connected TestForge to the real PHI organism modules (Brain, Orchestrator, Memory).
- **Homeostatic Observability:** Enabled real-time tracking of internal needs (Energy, Focus, Stress) during evaluation.
- **PEDI/DII Mapping:** Established causal links between TestForge perturbations and cognitive architecture state overrides.

### Phase 3: The Model Tournament (June 11, 2026)
- **Universal Scale-up:** Developed the `UniversalAdapter` for OpenAI-compatible API testing.
- **Massive Evaluation:** Executed the first **100-Model Tournament**, comparing the top industry systems across all cognitive dimensions.
- **Advanced Visualization:** Launched the comparative leaderboard and 100-model cognitive landscape plots.

## 🛠️ System Architecture (AFP)
The AFP operates on a three-tier cognitive testing model:
1. **Stimulus Layer:** Standardized multi-turn prompt suites (Consistency, Memory, Contradiction).
2. **Perturbation Layer:** Simulated stress conditions injected into the system's causal wiring.
3. **Metric Layer:** Statistical evaluation of behavioral and semantic stability.
